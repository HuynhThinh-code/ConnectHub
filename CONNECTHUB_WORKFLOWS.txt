CONNECTHUB - NOI DUNG SLIDE WORKFLOW QUAN TRONG
================================================

Muc tieu file:
- Dung lam noi dung dua vao slide thuyet trinh.
- Tap trung vao cac workflow quan trong cua web ConnectHub.
- Khong di qua qua nhieu chi tiet code nho.


SLIDE 1 - TONG QUAN HE THONG CONNECTHUB
---------------------------------------

ConnectHub la mot social media lab gom 3 phan chinh:

1. User Space
   - Dang ky, dang nhap
   - Dang bai, binh luan
   - Ket ban, nhan tin rieng
   - Cap nhat profile va avatar

2. Admin Space
   - Duyet bai / xoa bai
   - Quan ly user
   - Ban / unban user
   - Theo doi Intrusion Detection

3. Security Lab + AI Agent
   - Ghi nhan payload tan cong
   - Hien thi log theo thoi gian/IP/user
   - AI Fix / Learn Attack
   - ConnectHub Sentinel ho tro phan tich va de xuat cach fix

Workflow tong quan:

User/Admin
    -> Browser
    -> PHP Application
    -> MySQL Database
    -> Security Logs / AI Agent


SLIDE 2 - WORKFLOW REQUEST TONG THE
-----------------------------------

Moi request vao web deu di qua cac buoc:

Browser Request
    -> includes/db.php
    -> Ket noi MySQL
    -> Khoi tao session
    -> includes/security.php
    -> Kiem tra IP bi block
    -> Quet payload nghi ngo
    -> Kiem tra AI Fix rule
    -> Kiem tra user bi ban
    -> Chay logic cua page
    -> Render HTML / Tra JSON

Y nghia:
- Web khong chi xu ly chuc nang social.
- Moi request con duoc dua qua lop security monitoring.
- IDS co the ghi nhan SQLi, XSS, SSRF, command injection va unknown attack.


SLIDE 3 - WORKFLOW DANG NHAP VA PHAN QUYEN
------------------------------------------

Dang nhap:

User nhap username/password
    -> login.php nhan POST
    -> Kiem tra payload nghi ngo
    -> Query database users
    -> Tao session
    -> Tao session_token
    -> Luu token vao database
    -> Redirect theo role

Phan quyen:

Neu la Admin:
    -> Chuyen den admin.php

Neu la User:
    -> Chuyen den user.php

Neu chua login:
    -> Redirect ve login.php

Diem quan trong:
- Admin va User co trang rieng.
- Session token giup kiem tra phien dang nhap hien tai.
- Login activity duoc log lai trong admin dashboard.


SLIDE 4 - WORKFLOW USER SPACE
-----------------------------

Sau khi user dang nhap:

user.php
    -> Hien dashboard ca nhan
    -> Hien tong so bai viet
    -> Hien bai dang cho duyet
    -> Hien so ban be
    -> Quick access den:
        - Feed
        - Messages
        - Friends
        - Search
        - Settings

Workflow User chinh:

User Dashboard
    -> Feed: tao bai viet
    -> Profile: xem thong tin ca nhan/nguoi khac
    -> Friends: ket ban
    -> Messages: nhan tin rieng
    -> Settings: cap nhat profile/avatar

Y nghia:
- User Space la khu vuc thao tac cua nguoi dung binh thuong.
- Cac hanh dong cua user co the tao ra noi dung can admin duyet.


SLIDE 5 - WORKFLOW DANG BAI VA DUYET BAI
----------------------------------------

User dang bai:

User vao Feed
    -> Nhap noi dung bai viet
    -> Chon public/private
    -> Upload anh neu co
    -> Submit
    -> Luu vao bang posts voi status = pending
    -> Cho admin duyet

Admin moderation:

Admin vao admin.php
    -> Xem danh sach Pending Posts
    -> Approve / Reject / Delete
    -> Cap nhat status bai viet
    -> Ghi log hanh dong admin

Workflow:

User Post
    -> Database posts
    -> Pending queue
    -> Admin review
    -> Approved/Rejected/Deleted

Y nghia:
- Admin co vai tro kiem duyet noi dung.
- Bai viet khong tu dong public ngay neu dang o trang thai pending.


SLIDE 6 - WORKFLOW PROFILE, FRIENDS VA MESSAGES
-----------------------------------------------

Profile:

User mo profile.php?id=<user_id>
    -> Lay thong tin user
    -> Lay bai viet cua user do
    -> Hien nut Add Friend / Message

Friend request:

User bam Add Friend
    -> Tao friend_requests
    -> Nguoi nhan vao friends.php
    -> Accept hoac Reject
    -> Neu accepted thi tro thanh ban be

Private message:

User chon Chat
    -> messages.php?to=<id>
    -> Gui noi dung tin nhan
    -> Luu vao messages
    -> Poll tin nhan moi bang JavaScript/API

Workflow:

Profile
    -> Add Friend
    -> Friend accepted
    -> Message
    -> Realtime polling

Y nghia:
- Day la nhom chuc nang social chinh cua ConnectHub.
- Messenger co ca trang full messages va floating chat popup.


SLIDE 7 - WORKFLOW ADMIN SPACE
------------------------------

Admin Dashboard gom:

1. Moderation
   - Duyet bai
   - Tu choi bai
   - Xoa bai vi pham

2. User Management
   - Xem danh sach user
   - Ban user
   - Unban user

3. Security Monitoring
   - Xem login activity
   - Xem vulnerability attempts
   - Xem IP cua attacker
   - Xem payload

4. Response Action
   - Mark reviewed
   - Delete old event
   - Block IP
   - AI Fix
   - Learn Attack
   - Reset Lab State

Workflow:

Security/Event/Data
    -> Admin Dashboard
    -> Admin Review
    -> Action
    -> Log lai ket qua

Y nghia:
- Admin la trung tam dieu hanh va phong thu cua web.
- Tat ca hanh dong quan trong deu duoc ghi log.


SLIDE 8 - WORKFLOW INTRUSION DETECTION
--------------------------------------

Moi request vao web:

Request
    -> Collect URI, GET, POST, FILE name
    -> Quet regex payload
    -> Neu co dau hieu tan cong
    -> Ghi vao security_events
    -> Hien trong Admin Intrusion Detection

Loai su kien co the phat hien:

- SQL Injection
- XSS / Stored Script
- Path Traversal
- Command Injection
- SSRF
- OAuth Scope Escalation
- Unknown Attack
- AI Learned Attack

Thong tin log:

- Thoi gian
- User / Guest
- IP address
- Event type
- Severity
- URL / Route
- Payload
- Details

Y nghia:
- IDS giup admin thay ai dang tan cong, tan cong luc nao, payload la gi.
- Day la phan quan trong de viet bao cao kiem thu bao mat.


SLIDE 9 - WORKFLOW AI FIX
-------------------------

Khi IDS phat hien loi:

Security Event
    -> Admin thay trong Intrusion Detection
    -> Admin bam AI Fix
    -> Tao rule trong ai_fix_rules
    -> Event chuyen sang FIXED
    -> Ghi memory vao ai_agent_memory

Khi attacker thu lai:

Request moi
    -> Kiem tra ai_fix_rules
    -> Neu payload match rule
    -> Chan request hoac dua qua code path an toan
    -> Ghi log AI Fix blocked/handled

Workflow:

Attack Detected
    -> AI Fix
    -> Rule Active
    -> Future Attack Blocked

Y nghia:
- AI Fix la co che bao ve tam thoi/huong dan fix.
- Lab van co the reset lai de test lai cac lo hong.


SLIDE 10 - WORKFLOW CONNECTHUB SENTINEL
---------------------------------------

ConnectHub Sentinel la AI Agent trong Admin Dashboard.

Chuc nang:

- Admin hoi cach fix loi
- Agent doc security_events gan day
- Agent doc ai_agent_memory
- Agent scan code hotspots
- Agent goi Gemini API
- Agent tra loi cach fix
- Agent co the de xuat patch source code

Workflow chat:

Admin question
    -> api/ai_agent.php
    -> Load recent logs
    -> Load agent memory
    -> Load code hotspots
    -> Call Gemini model
    -> Return answer
    -> Render chat bubble

Workflow patch:

AI response co patch block
    -> UI hien Apply Source Patch
    -> Admin bam apply
    -> Tao file .bak
    -> Ghi code moi vao source file
    -> Log ai_hotpatch_applied
    -> Luu AI memory

Y nghia:
- Sentinel khong chi tra loi text.
- No co the chi vi tri code nghi ngo va de xuat cach sua.


SLIDE 11 - WORKFLOW LEARN UNKNOWN ATTACK
----------------------------------------

Lan dau gap tan cong la:

Unknown payload
    -> IDS khong match pattern cu
    -> Neu payload co dau hieu bat thuong
    -> Log unknown_attack
    -> Admin thay event moi

Admin cho agent hoc:

Admin bam Learn Attack
    -> Sentinel lay payload + route
    -> Tinh fingerprint
    -> Scan code hotspot
    -> Goi Gemini de phan loai neu co API key
    -> Neu API loi thi dung heuristic fallback
    -> Luu vao ai_attack_memory

Lan sau gap lai:

Request co fingerprint giong
    -> IDS tim trong ai_attack_memory
    -> Log learned_attack
    -> Tang seen_count

Workflow:

Unknown Attack
    -> Learn Attack
    -> Save Attack Memory
    -> Next Time Recognized

Y nghia:
- Agent co kha nang hoc pattern moi.
- Lan dau chua biet, lan sau co the nhan dien lai.


SLIDE 12 - WORKFLOW RESET LAB STATE
-----------------------------------

Muc dich:
- Dua lab ve trang thai ban dau de test lai lo hong.

Khi admin bam Reset Lab State:

Confirm reset
    -> Xoa AI fix rules active
    -> Xoa security logs
    -> Xoa blocked IPs
    -> Reset users/posts/comments/messages/friends
    -> Seed lai du lieu ban dau
    -> Xoa upload phat sinh
    -> Restore source tu file .bak neu co
    -> Tao lai session admin

Ket qua:

Lab State
    -> Quay lai ban dau
    -> Co the khai thac/test lai loi

Ghi chu:
- Reset Lab State khong phai nut xoa project.
- No phuc vu viec demo lai quy trinh tan cong -> phat hien -> fix.


SLIDE 13 - WORKFLOW CAC LO HONG TRONG LAB
-----------------------------------------

ConnectHub co cac lo hong duoc cai san de demo:

1. SQL Injection
   - login.php
   - search.php

2. XSS
   - search.php
   - index.php
   - post.php
   - messages.php

3. IDOR / Private Disclosure
   - profile.php
   - messages.php
   - api/get_messages.php

4. Weak Session
   - includes/db.php
   - login.php

5. Command Injection / Upload Bypass
   - settings.php

6. SSRF
   - preview.php

7. OAuth Scope Escalation
   - oauth.php

Workflow demo loi:

Attacker payload
    -> Vulnerable route
    -> IDS log event
    -> Admin review
    -> AI Fix / Report

SLIDE 15 - TOM TAT CAC WORKFLOW CHINH
-------------------------------------

1. User workflow:
   Register/Login -> User Space -> Feed/Profile/Friends/Messages/Settings

2. Admin workflow:
   Login -> Admin Dashboard -> Moderate Content -> Manage Users -> Monitor Security

3. Security workflow:
   Request -> IDS scan -> security_events -> Admin review

4. AI workflow:
   Event -> AI Fix / Learn Attack -> Memory -> Future detection

5. Reset workflow:
   Reset Lab State -> Seed data -> Restore vulnerable lab -> Demo lai duoc

Thong diep chinh:

ConnectHub khong chi la social web.
No la mot lab bao mat co day du:
- Chuc nang nguoi dung
- Chuc nang admin
- Lo hong de khai thac
- IDS de quan sat
- AI Agent de hoc va de xuat fix
